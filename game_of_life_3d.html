📘 How the 3D rules work — click to expand

The neighborhood

Every voxel (x,y,z) in the cubic grid has a Moore neighborhood of 26 neighbors (the 3³ − 1 cells touching it on faces, edges, and corners). With wrap (T³) on, the grid is a 3-torus — the right face wraps to the left, top to bottom, front to back. With wrap off, neighbors past the wall count as dead.

The rule string: B<born>/S<survive>

This is the 3D analog of Conway's B3/S23. For each voxel, we count its n live neighbors (0–26):

  • If the voxel is dead and n ∈ B set → it is born next step.
  • If the voxel is alive and n ∈ S set → it survives.
  • Otherwise → dead next step.

Counts are comma-separated because in 3D you regularly need two-digit numbers. Examples:

RuleNameBehavior
B6/S5,6,7Bays 5766 (default)Canonical "3D Life" — has gliders, oscillators, still lifes.
B5/S4,5Bays 4555Bays' second well-studied rule; gliders + spaceships.
B4,5/S5—Sparse, crystal-like growth.
B3/S4,5,6,7—Explosive, tends to fill space.
B2/S3D "seeds"Pure birth, never survives — fast diffusing front.

To change the rule: type a new string into the Rule textbox and press Tab or Enter. Spaces are ignored. Both B6/S567 and B6/S5,6,7 work for single-digit counts; use commas for 10+ (B6/S10,11).

Color rule (independent of the life/death rule)

Each live voxel carries a hue ∈ [0°, 360°). The Color dropdown decides what newborn voxels get:

  • position (azimuth) — hue from atan2(y−c, x−c) (angle around the central z-axis). The grid behaves like a fixed colored field; "motion" is just births sampling that field.
  • lock at birth — newborn hue from its position, then frozen until death. Each pattern keeps its birthplace tint.
  • inherit from parents (default) — newborn hue = circular mean of its live neighbors' hues, i.e. atan2(Σ sin h, Σ cos h). Color advects along the discrete flow, so gliders visibly carry their color across the grid.

Survivors always keep their previous hue.

Controls

  • drag rotate · shift+drag pan · wheel zoom
  • Space run/pause · S step · R random · C clear · Enter start · F fullscreen
  • Init JSON loads { grid:[Lx,Ly,Lz], points:[{x,y,z,hue?}], rule?, wrap? }. Out-of-grid points are bbox-fit; multiple points per voxel get circular-mean hue.
  • Save JSON dumps the current state in the same format.

Performance budget

  • 96³ — borderline; consider Step
  • ≥ 128³ — Step-only (160³ is the hard cap)
